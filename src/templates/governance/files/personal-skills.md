# 个人 Skills 兼容规则

团队允许每个人保留自己的 skills、快捷指令、提示词模板和本地工作流。

这类能力只影响个人操作方式，不改变仓内正式规范。

## 不可被覆盖的事项

- 当前 `change` 范围
- `.specnfc/` 下的模块规则
- `spec / plan / tasks / decisions / status / release-handoff` 正式文件结构
- 最低验证要求
- 裁决点、安全边界和发布由人接手的边界

## 可以保留的个人习惯

- 自己常用的思考顺序
- 自己常用的检索、编辑、测试方式
- 自己常用的提示词风格
- 自己额外增加的本地检查动作

## 冲突处理

1. 先按 `AGENTS.md`、`CLAUDE.md` 或 `.trae/rules/project_rules.md` 执行。
2. 再按 `.specnfc/` 和当前 `change` 的正式文件执行。
3. 个人 skill 如果要求额外过程文件，只能留在本地，不能当正式交付。
4. 无法判断时，把冲突和处理结论写回 `status.md` 或 `decisions.md`。

## 交接要求

- 下游只接正式文件。
- 不接聊天记录。
- 不接私有提示词。
- 不接个人 skill 专用过程产物。

## 开场口径

可直接使用下面这句作为开场：

```text
按仓内 Spec nfc 规范执行。本次保留个人 skills 习惯，但如与仓内规则冲突，以入口文件、.specnfc/ 和当前 change 为准，只交付正式文件。
```
