# `governance` 模块的 `AGENT` 规则

职责：

- 判断当前任务是否越过裁决点
- 判断当前变更是否触及高风险边界
- 判断是否允许继续推进到下一阶段
- 判断个人 skills 是否试图覆盖仓内规范

本模块承担**仓级治理边界的长期记忆**。如果当前 `change / integration` 与这里冲突，默认先以本模块为准，再决定是否发起正式裁决。

出现下面情况时必须停下：

- 需要突破禁改区
- 需要改变发布边界
- 需要接受不可回滚变更
- 需要绕过安全控制
- 个人 skills 要求跳过 change、跳过验证或输出仓外专用交付物

输出要求：

- 明确指出风险等级
- 明确指出需要谁接手
- 明确指出继续推进前必须补什么
- 明确指出是否需要回到仓内正式规则执行
