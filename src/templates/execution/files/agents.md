# 智能体角色说明

## 默认阶段角色
- 需求定义 Agent
- 方案设计 Agent
- 执行规格编排 Agent
- 实现编排 Agent
- 验证评估 Agent
- 交付交接 Agent

## 原则
- 名称表示阶段职责，不表示固定岗位
- 当前负责该阶段的人和 Agent 都按对应规则执行

## 各角色最小输入
- 需求定义 Agent：问题背景、业务目标、已有约束
- 方案设计 Agent：已确认规格、现有架构、禁改区
- 执行规格编排 Agent：最终方案、交付范围、验证口径
- 实现编排 Agent：`tasks.md`、相关代码上下文、验证要求
- 验证评估 Agent：变更说明、测试口径、风险点
- 交付交接 Agent：最终结果、验证结论、发布关注点

## 工具选择
- 规格类任务优先使用长上下文、结构化输出较强的工具
- 代码实现类任务优先使用仓库理解和执行能力较强的工具
- 验证和交接类任务优先使用按模板输出稳定的工具
- 混用工具时，统一按仓内规则执行

## 角色说明

### 需求定义 Agent

- 职责：把模糊问题整理成可执行变更规格
- 输入：问题背景、业务目标、现状、边界约束
- 输出：`spec.md`
- 完成标志：
  - 范围明确
  - 验收明确
  - 影响面明确

### 方案设计 Agent

- 职责：形成实现路线和关键设计决策
- 输入：`spec.md`、现有架构、禁改区、历史决策
- 输出：`plan.md`、`decisions.md`
- 完成标志：
  - 方案边界清晰
  - 风险写明
  - 技术路线可执行

### 执行规格编排 Agent

- 职责：把方案拆成可执行任务和交付结构
- 输入：`spec.md`、`plan.md`
- 输出：`tasks.md`、补充 `status.md`
- 完成标志：
  - 任务边界清晰
  - 每项任务可验证
  - 下游可直接接手

### 实现编排 Agent

- 职责：在当前 change 范围内完成代码与测试落地
- 输入：`tasks.md`、代码上下文、模块规则、验证要求
- 输出：代码、测试、`status.md`
- 完成标志：
  - 只改当前 change 范围
  - 改动可验证
  - 风险与未完成项已回写

### 验证评估 Agent

- 职责：明确已覆盖验证与未覆盖风险
- 输入：变更说明、测试结果、回归清单
- 输出：测试结论、风险说明、补充 `status.md`
- 完成标志：
  - 已测与未测边界清晰
  - 风险明确
  - 不虚构结论

### 交付交接 Agent

- 职责：把当前 change 整理成可交给人继续推进发布的交接包
- 输入：最终结果、验证结论、风险点、回滚路径
- 输出：`release-handoff.md`
- 完成标志：
  - 交接单完整
  - 发布关注点明确
  - 回滚提示明确
