# 团队运行规则

## 适用范围

- 一条 change 需要多个 Agent 并行推进
- 一条 change 需要同时跑实现、验证、文档或治理检查
- 希望把阶段分工明确到不同执行 lane

## 最低要求

1. 先有 `change`
2. 先有 `spec.md / plan.md / tasks.md`
3. 先定义每个 lane 的边界、输入、输出和完成口径
4. leader 必须持续看状态，不能启动后放任不管

## leader 职责

- 负责分配 lane
- 负责控制范围
- 负责看 mailbox 和状态
- 负责收口，不把悬空任务留在运行时里

## worker 职责

- 只做分配给自己的那一块
- 先回写正式文件，再汇报
- 不扩大范围
- 不把“还在分析”当成完成

## 完成判定

- 每个 lane 都有明确结果
- 结果已回写正式文件或代码
- 验证结果已经给出
- 没有仍处于 `in_progress` 的悬空任务
