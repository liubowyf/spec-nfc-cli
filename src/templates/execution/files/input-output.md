# 输入输出规则

## 必带输入
- 当前 change 文档
- 相关代码上下文
- 当前边界与禁改区
- 当前最低验证要求

## 补充输入
- 相关历史决策
- 同类功能已有实现
- 当前仓模块启用状态
- 需要遵守的接口或数据约束

## 输出要求
- 只保留最终结果
- 只保留必要上下文
- 不传完整过程流水账

## 标准结构
- Summary：最终完成了什么
- Changes：具体改动对象
- Risks：遗留风险和高风险点
- Validation：已执行验证
- Next：下游接续动作

## 输入包

```text
1. 当前 change 目录
2. 当前仓相关模块目录
3. 相关代码与测试
4. 当前任务目标
5. 当前最低验证要求
```

## 输出包

```text
1. 正式文件更新
2. 改动摘要
3. 验证结果
4. 风险与未完成项
5. 后续动作
```
