# 团队恢复手册

## 什么时候判定为卡住

- 长时间没有新的协作消息
- 任务一直停在 `in_progress`
- 执行端只有 queued message，没有新的结果
- 协调端持续收到 stalled nudge

## 协调者恢复顺序

1. 先看 `team status`
2. 再看主控 inbox
3. 再看执行端 inbox
4. 再看 pane 尾部

## 常见恢复动作

- 收紧范围
- 指定只交付最小修复
- 指定先汇报再继续
- 指定关键路径执行端先完成
- 明确“当前仓不是 git repo，commit 不作为阻塞条件”

## 终止条件

如果执行端已有有效改动落盘，但运行时仍无法收口：

1. 保留已落盘改动
2. 强制结束执行端 pane
3. 清理 team 状态
4. 由协调者本地整合和验证
