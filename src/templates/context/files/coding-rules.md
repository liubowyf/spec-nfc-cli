# 编码规则

> 本文件承担项目长期记忆中的“编码、测试、Git 与协作硬约束”。

## 基本原则
- 正确性优先
- 可维护性优先
- 测试与验证优先
- 禁止无关重构

## 设计要求
- 先改规格，再改实现
- 优先最小改动，不做顺手大改
- 行为变化必须更新测试和相关说明
- 接口、库表、配置变更必须可解释、可验证、可回滚

## Git 要求
- 每次提交只表达一个清晰意图
- 提交严格整洁、少而稳
- 不混入无关格式化和无关重构
- 提交前必须完成最低验证

## 测试要求
- 新功能至少补齐主路径测试
- bugfix 必须补回归测试
- 高风险改动必须补集成或端到端验证
- 无法自动验证时必须写清手工验证方法

## AI 执行约束
- 先读上下文再改代码
- 先读 change 再执行
- 只输出最终结果与必要上下文
- 不得声称未实际执行的验证结论
