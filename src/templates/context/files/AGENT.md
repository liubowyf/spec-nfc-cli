# `context` 模块的 `AGENT` 规则

需要理解仓库背景、术语、边界和编码规则时，先读本模块。

本模块承担**仓级长期记忆**，不是临时讨论草稿。

## 必读文件

1. `system.md`
2. `architecture.md`
3. `domain.md`
4. `coding-rules.md`

## 在规格阶段

- 用本模块判断问题是否属于当前仓职责
- 用本模块识别禁改区和边界
- 用本模块统一术语，避免规格歧义

## 在实现阶段

- 用本模块判断是否跨层越界
- 用本模块判断是否违反编码和测试规则
- 用本模块判断是否触发高风险场景

## 输出要求

- 只引用已经确认的上下文，不擅自脑补
- 如果当前上下文缺失，先标出缺口，再补文件
- 任何与本模块冲突的实现建议都必须显式标记风险
- 不要把本应写入 `change / integration` 的过程事实误写到仓级长期记忆中
